# App Store — "What's New in This Version" (4000 char max)

App Store Connect → My Apps → DealGapIQ → [Version 2.2.0] → iOS App → **What's New in This Version**

**Fill this in — it is expected for an update** (it only hides on a brand-new
app's first version). It does NOT increase review risk; the only release-notes
mistake that causes rejection is claiming features that don't exist. This is
also user-facing: existing users read it in the Updates tab.

> **Rules of the road**
> - Describe changes since the **last live version**, not since the original
>   submission. If several builds went unreleased, summarize the net-new.
> - Only list functionality that **actually ships in this build** — every claim
>   is testable by the reviewer.
> - Version-locked: unlike Promotional Text, this can't be edited after the
>   version goes live without submitting a new version.

> **Important:** the bullets above reflect DealGapIQ's full feature set. Keep

## Why the closing line matters

"We read every review" gently nudges satisfied users toward leaving a rating —
which reinforces the in-app review prompt and helps lift the star count that
gates both organic ranking and paid conversion.
